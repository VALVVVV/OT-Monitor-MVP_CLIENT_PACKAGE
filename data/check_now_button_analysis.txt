Анализ кнопки «Проверить сейчас» в MVP

Где находится логика в app.js:
- кнопка берётся через `const checkButton = document.getElementById("check-now")`
- обработчик назначается в `setupActions()`
- сам обработчик находится в блоке `checkButton.addEventListener("click", () => { showCheckNotification(); })`
- основная логика выполняется в функциях:
  - `showCheckNotification()`
  - `runDemoSearchCheck()`
  - `addDemoDocumentsIfMissing(datetime)`

Является ли кнопка реальной проверкой:
- нет
- это демонстрационная имитация
- запусков Python-скриптов, backend-команд, `exec`, `spawn` или вызовов внешнего API для мониторинга не найдено

Что происходит после нажатия:
- вызывается `showCheckNotification()`
- затем вызывается `runDemoSearchCheck()`
- в журнал проверок интерфейса добавляются демонстрационные записи по 3 темам:
  - Минтруд России — «Обучение по охране труда»
  - Минздрав России — «Медицинские осмотры»
  - МЧС России — «Огнетушители»
- после этого вызывается `addDemoDocumentsIfMissing(datetime)`
- если демонстрационные документы ещё не были добавлены в текущий массив `documents`, они добавляются в начало списка через `documents.unshift(doc)`
- затем интерфейс перерисовывает список документов, статистику и журнал проверок

Какие именно документы добавляются:
- демонстрационное изменение по обучению охране труда
- демонстрационное изменение по медицинским осмотрам
- документ по огнетушителям в этой логике НЕ добавляется
- для темы «Огнетушители» в журнале создаётся запись с `newDocs = 0`

Добавляются ли записи в журнал проверок:
- да, но только во встроенный UI-журнал `checkLog`
- используется `addCheckLogEntry(entry)`, которая делает `checkLog.unshift(entry)`
- это изменение живёт в памяти страницы

Пишутся ли изменения в data/documents.json:
- нет
- `data/documents.json` только читается при инициализации через `fetch("data/documents.json", { cache: "no-store" })`
- записи обратно в файл не выполняются

Пишутся ли изменения в data/check_log.json:
- нет
- чтения `data/check_log.json` и записи в него в app.js не найдено
- журнал проверок экрана формируется из встроенного массива `checkLog`

Используется ли localStorage:
- да, но не для кнопки «Проверить сейчас»
- `localStorage` используется для:
  - статусов/решений по документам (`otmonitor_decision_*`)
  - журнала решений инженера (`otmonitor_decision_log`)
- для списка документов, источников и журнала проверок кнопки `localStorage` не используется

Сохраняются ли добавленные документы после обновления страницы:
- нет
- добавление происходит только в оперативный массив `documents` внутри текущей сессии страницы
- после обновления страницы данные снова загружаются из `data/documents.json`
- поэтому демонстрационно добавленные этой кнопкой документы пропадают после перезагрузки

Какие данные кнопка меняет:
- временно меняет массив `documents` в памяти страницы
- временно меняет встроенный массив `checkLog` в памяти страницы
- показывает уведомление в интерфейсе

Какие файлы кнопка НЕ меняет:
- `data/documents.json`
- `data/check_log.json`
- `data/sources.json`
- `app.js`
- `index.html`
- `style.css`

Безопасный следующий шаг:
- не использовать кнопку «Проверить сейчас» как доказательство реального мониторинга
- для демо-контура либо явно пометить эту кнопку как демонстрационную
- либо позже перевести её на запуск реальной цепочки чтения JSON-результатов от подготовленных Python-скриптов, не смешивая это с встроенной имитацией
