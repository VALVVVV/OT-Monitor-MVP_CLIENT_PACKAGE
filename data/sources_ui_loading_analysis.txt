Анализ загрузки источников мониторинга в интерфейсе MVP

Откуда сейчас берутся источники:
- раздел «Источники» формируется в app.js из встроенного массива sources
- список рендерится функцией renderSourcesList(), которая делает sources.forEach(...)
- data/documents.json загружается отдельно только для списка документов и не управляет списком источников

Используется ли data/sources.json:
- нет
- в app.js не найдено загрузки data/sources.json через fetch
- поэтому data/sources.json может быть пустым, но в интерфейсе всё равно отображаются источники

Есть ли встроенный массив источников внутри app.js:
- да
- в app.js объявлен const sources = [...]
- там сейчас зашиты источники для интерфейса

Используется ли localStorage для источников:
- нет
- localStorage используется для решений по документам и журнала решений
- признаков хранения списка источников в localStorage не найдено

Почему data/sources.json пустой, но в интерфейсе что-то отображается:
- потому что интерфейс не читает data/sources.json
- вместо этого он показывает встроенный список sources из app.js
- дополнительно в app.js есть и другие встроенные справочные массивы, например schedules и searchQueries

Какие файлы нужно будет менять позже, если приводить источники к нормальному виду:
- app.js
- data/sources.json

Что именно, вероятно, придётся сделать позже:
- вынести список sources из app.js в data/sources.json
- добавить в app.js загрузку data/sources.json через fetch
- при необходимости отдельно решить, что делать с встроенными schedules и searchQueries, потому что они тоже сейчас зашиты в app.js

Безопасный следующий шаг:
- сначала оставить текущий интерфейс без изменений
- затем отдельно подготовить новую структуру data/sources.json для демонстрационного контура
- после этого аккуратно доработать только app.js, чтобы он загружал sources из JSON с безопасным fallback на встроенные данные
