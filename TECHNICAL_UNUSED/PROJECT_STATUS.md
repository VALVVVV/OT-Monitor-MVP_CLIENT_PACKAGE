# PROJECT_STATUS.md

## 1. Текущий статус проекта

Проект: ОТ-Монитор MVP.

Текущая версия: v0.1.

Это локальный демонстрационный MVP системы информирования инженера по охране труда о нормативных изменениях.

Проект запускается простым открытием файла index.html в браузере.

Сервер, база данных, авторизация, React, Node.js, npm и Vite не используются.

## 2. Что уже реализовано

В текущей версии реализованы основные демонстрационные блоки:

* Главная страница MVP;
* Источники мониторинга;
* Расписание проверок;
* Настройки поиска;
* Кнопка "Проверить сейчас";
* Журнал проверок;
* Лента найденных документов;
* Карточка выбранного документа;
* Статус обработки документа;
* Комментарий инженера;
* Журнал решений инженера;
* Сохранение решений через localStorage;
* Блок будущего ИИ/RAG;
* Блок черновика рекомендации.

## 3. Рабочая цепочка MVP

Сейчас работает демонстрационная цепочка:

1. Пользователь видит источники мониторинга.
2. Пользователь видит расписание проверок.
3. Пользователь видит настройки поиска.
4. Пользователь нажимает кнопку "Проверить сейчас".
5. Система имитирует проверку по нескольким настройкам поиска.
6. В журнал проверок добавляются записи.
7. В ленту найденных документов добавляются демонстрационные документы.
8. Пользователь открывает карточку найденного документа.
9. Пользователь меняет статус обработки.
10. Пользователь пишет комментарий инженера.
11. Пользователь нажимает "Сохранить решение".
12. Решение сохраняется.
13. Запись появляется в журнале решений инженера.
14. После обновления страницы сохранённые решения восстанавливаются из localStorage.

## 4. Что пока имитируется

В текущей версии пока имитируется:

* реальная проверка сайтов;
* автоматическая проверка по расписанию;
* скачивание и сохранение копий нормативных документов;
* анализ текста нормативного документа;
* ИИ/RAG-обработка;
* подготовка черновых рекомендаций.

Кнопка "Проверить сейчас" пока не обращается к реальным сайтам. Она имитирует проверку по тестовым настройкам поиска.

## 5. Что реально работает сейчас

Реально работает:

* отображение интерфейса;
* выбор документа из ленты;
* отображение карточки документа;
* добавление строк в журнал проверок;
* добавление демонстрационных найденных документов в ленту;
* защита от дублей демонстрационных документов;
* изменение статуса документа;
* ввод комментария инженера;
* сохранение решения в localStorage;
* восстановление решений после обновления страницы;
* отображение журнала решений инженера.

## 6. Основные файлы проекта

index.html — структура страницы и основные блоки интерфейса.

style.css — внешний вид страницы, карточек, таблиц и блоков.

app.js — тестовые данные и логика работы интерфейса.

PROJECT_CONTEXT.md — описание смысла проекта и ограничений для Codex.

PROJECT_STATUS.md — текущий статус проекта и контрольная точка разработки.

## 7. Важные ограничения

Проект должен оставаться простым:

* только HTML, CSS, JavaScript;
* без сервера;
* без базы данных;
* без авторизации;
* без React;
* без Node.js;
* без npm;
* без Vite;
* без package.json.

Не нужно самовольно усложнять архитектуру.

## 8. Следующие возможные шаги

Возможные следующие этапы:

1. Улучшить структуру тестовых данных.
2. Перенести тестовые данные из app.js в JSON-файлы из папки data.
3. Сделать более понятное меню или навигацию по блокам.
4. Добавить фильтры по разделам и статусам.
5. Добавить отдельный экран или блок "Типы внутренних документов".
6. Подготовить первый простой вариант реальной проверки одного источника.
7. Продумать безопасную архитектуру будущего реального мониторинга.
8. Подготовить демонстрационный сценарий показа MVP клиенту.

## 9. Главный принцип дальнейшей разработки

Сначала сохранять простую рабочую версию.

Каждую новую функцию добавлять маленькими шагами:

1. поставить задачу;
2. изменить минимум файлов;
3. проверить в браузере;
4. принять изменения;
5. зафиксировать результат.

Главная ценность MVP — показать рабочий процесс:

источники → расписание → настройки поиска → проверка → найденный документ → карточка → решение инженера → журнал решений.
