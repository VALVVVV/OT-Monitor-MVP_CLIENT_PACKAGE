# ОТ-Монитор MVP — план следующего этапа

## 1. Текущий статус проекта

* MVP является рабочей локальной версией.
* Проект находится в папке `D:\OT-Monitor-MVP`.
* Стабильная копия создана:
  `D:\OT-Monitor-MVP_STABLE_2026-06-11`
* Стабильную копию не изменять.
* Дальнейшую разработку вести только в `D:\OT-Monitor-MVP`.

## 2. Что уже работает

* MVP запускается через `start_mvp.bat`.
* `start_mvp.bat` запускает `server.py` и открывает браузер.
* `server.py` поднимает локальный сервер на `http://127.0.0.1:8000`.
* Кнопка «Проверить сейчас» в интерфейсе реально запускает backend через `POST /api/run-demo-check`.
* `run_demo_check.py` запускает проверку трёх источников:
  * Минтруд России: `https://mintrud.gov.ru/docs`
  * МЧС России: `https://mchs.gov.ru/dokumenty/vse-dokumenty`
  * Минздрав России: `https://minzdrav.gov.ru/documents`
* После проверки обновляются:
  * `data/documents.json`
  * `data/check_log.json`
* На главной странице обновляется информация о последней проверке.
* Журнал проверок показывает новые записи.
* Решения инженера, статусы и комментарии сохраняются, если используются текущим интерфейсом.

## 3. Рабочая цепочка MVP

```text
start_mvp.bat
→ server.py
→ app.js
→ POST /api/run-demo-check
→ run_demo_check.py
→ inspect_*.py
→ merge_*.py
→ data/documents.json + data/check_log.json
→ обновление интерфейса MVP
```

## 4. Важные технические условия

* Для демонстрации лучше отключать VPN.
* Без VPN все 3 источника ранее проходили успешно.
* С VPN ранее были проблемы у Минтруда и Минздрава.
* МЧС работал и с VPN.
* Если внешний сайт долго не отвечает, возможна ошибка `ReadTimeout`.
* Это нормальный риск для MVP, его нужно пояснять пользователю.

## 5. Принятая развилка по дальнейшей работе

Следующий этап — не дизайн и не клиентский пакет.

1. Функциональная логика экранов.
2. Понятность интерфейса.
3. Внешний дизайн.

## 9. Как работать с пользователем

Пользователь не программист.
Нужно работать короткими шагами.
Перед изменениями давать план.
Не давать сразу много команд.
Всегда пояснять простыми словами:

* `server.py` — локальный серверный файл;
* `app.js` — файл логики интерфейса;
* `start_mvp.bat` — файл запуска MVP одним кликом;
* endpoint — адрес команды внутри сервера;
* JSON — текстовый файл с данными;
* backup — резервная копия перед изменением.

Перед изменением важных файлов всегда создавать backup.
